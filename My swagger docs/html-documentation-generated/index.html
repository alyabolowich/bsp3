EU emissions embodied in trade

Have you ever wondered about the hidden environmental and social impacts of globalization? How much CO2 or vulnerable labor 'imported' as a result of trade between two countries? And of that trade, what economic sector is responsible for the most damage? Or the least? Having answers to these questions can help us to better understand our consumption and production impacts at a global scale.

This is an API to source data regarding emissions embodied in trade in the EU. The primary use of this API is for those interested to learn about the environmental and social impacts of supply chains. Using this API, it is possible to find out the quantity of impacts (environmental, like CO2, or social, like vulnerable labor) that sectors and/or countries produce and consume.

NOTE: If there is a problem connecting to the database (such as an incorrect password), this will return a message corresponding to a 500 error saying: FATAL: password authentication failed for user. If the URLs are incorrectly formatted, this will return a message corresponding to a 404 error (example shown in the region and sector endpoints).

Stressors

Up 
get /stressors/{stressorID}/exports
Find region exports to other regions (getExports)
Select a country for which you wish to know its exports. This returns the sum of the region's stressors exported to all other regions. The region entered should be the region for which you want to see exports. For example, region=de will return the total stressor value that Germany exports to all other regions in the dataset.

Path parameters

stressorID (required)
Path Parameter — For CO2 emissions, enter co2. For vulnerable labor, enter vl. For the economy, enter economy.

Query parameters

region (required)
Query Parameter — Region exporting the stressor

Return type

Exports

Example data

{
  "result" : [ {
    "sum" : 3.6370659886E8,

  }, {
    "sum" : 3.6370659886E8,
    "region_from" : "sk"

get /stressors/{stressorID}
Find region-to-region or sector-to-sector stressors (stressorsStressorIDGet)
This endpoint can be queried to identify individual sectors and/or regions that either export or import stressors from other sectors or regions. The user must input at minimum either a sector or a region and at maximum can select outgoing and ingoing regions and sectors. The user will be notified of errors in case of bad events: 1) 400 error bad request if no query parameters are selected or 2) 400 error for incorrect region/sector input.

NOTE: You must enter at least one of the query parameters!

Path parameters

stressorID (required)
Path Parameter — For CO2 emissions, enter co2. For vulnerable labor, enter vl. For the economy, enter economy.

Query parameters

region_to (optional)
Query Parameter — Region importing the stressor
region_from (optional)
Query Parameter — Region exporting the stressor
sector_to (optional)
Query Parameter — Sector importing the stressor
sector_from (optional)
Query Parameter — Sector exporting the stressor

Return type
